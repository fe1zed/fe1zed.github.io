Instant Localization Documentation

v1.2.3 · Unity 2021.3+

Requirements

  • Unity 2021.3 LTS or newer (recommended)
  • TextMeshPro package (required for TextMeshProUGUI)
  • Localization CSV files saved as UTF-8

The Instant Localization asset provides a quick and efficient way to add multilingual support to your Unity project, primarily targeting TextMeshProUGUI components. The system relies on CSV files stored in the Resources folder and enables static method calls for language management.

  • CSV-based localization supporting multiple languages
  • Runtime language switching with automatic UI refresh
  • TextMeshPro integration via LocalizeText component
  • Localization Key Selector Window with search and filtering
  • Built-in Favorites system
  • Real-time key management across language files
  • AI-powered Auto-Translate Window (DeepL & Azure)
  • Translation preview and caching

Project Setup

  1. Import the TextMeshPro package into your project
  2. Navigate to Assets/Instant Localization and locate the Resources folder (create if missing)
  3. Place your CSV language files directly into Assets/Instant Localization/Resources/

The filename (without the .csv extension) is used as the language code when switching languages:

  • English.csv → language code "English"
  • German.csv → language code "German"
  • Spanish.csv → language code "Spanish"

Every CSV file must have a header row followed by key/value pairs. Keys must be single identifiers without spaces:

key,text
examine,Examine
save_game,Save Game
gamepadSensitivity,Gamepad Sensitivity
example_complex,{0}, your character is level {1}. You have {2} skill points and your current quest is: {3}.

Keys must be identical across all language files. The system returns the key itself as a fallback if a translation is missing.

LocalizeText Component

To localize a UI text element:

  1. Create a UI Text object with a TextMeshProUGUI component
  2. Add the LocalizeText script to the same GameObject
  3. Set the key in the Inspector using the Localized Text ID field

You can assign a key two ways:

  • Click the Search button to open the Key Selector Window and pick a key visually
  • Type a key manually — it must exist in the CSV files

The component automatically looks up the key for the currently active language and displays the corresponding text. It subscribes to LocalizationSystem.OnLanguageChanged and refreshes instantly — in both Editor and runtime — whenever the language changes.

Key Selector Window

Open the window from the menu or via shortcut:

Tools → Localization → Open Localization Window  // Shortcut [Ctrl+Shift+T]

Or click the Search button inside any LocalizeText component in the Inspector.

Language Selection — switch the preview language via the dropdown to see translations side by side.

Search & Filter — filter keys by name or value; matching text is highlighted. Toggle Sort A–Z for alphabetical ordering. The CSV button opens the current file in your default app.

Favorites — click the star icon next to any key to mark it as a favourite. Favourites always appear at the top of the list and persist via a ScriptableObject at Assets/Instant Localization/Data/InstantLocalizationFavorites.asset.

Selecting a key:

  • Single-click to select and preview
  • Double-click to apply the key and close the window
  • Click Select to apply without closing

Managing Keys

All key operations update every language file simultaneously — no need to edit CSVs by hand.

Adding a new key:

  1. Click Add New
  2. Enter the key name and translations for each language
  3. Click Create — the key is added to all CSV files

Editing a key:

  1. Select the key from the list
  2. Click Edit
  3. Modify translations for any language
  4. Click Save

Deleting a key:

  1. Select the key from the list
  2. Click Delete and confirm — removes from all CSV files

Manage keys through the Key Selector Window rather than editing CSV files directly. This ensures consistency across all language files and prevents mismatched entries.

Switching Languages at Runtime

The active language for the entire application is controlled by the static LocalizationSystem class. Call SetLanguage with a language code that matches a CSV filename:

// Switches to Spanish — loads 'Spanish.csv' and updates all UI texts.
fe1zed.InstantLocalization.LocalizationSystem.SetLanguage("Spanish");

All LocalizeText components refresh automatically. You can also read the current language:

string current = fe1zed.InstantLocalization.LocalizationSystem.Language;

Localize via Code

Use GetLocalisedValue for dynamic or formatted text. The method supports string interpolation via ordered placeholders — the first argument replaces {0}, the second replaces {1}, and so on.

The package includes a fully commented example at Assets/Instant Localization/Example/Example.cs:

using UnityEngine;
using TMPro;

namespace fe1zed.InstantLocalization
{
    [RequireComponent(typeof(TextMeshProUGUI))]
    // [ExecuteAlways] // uncomment to preview directly in Editor
    public class Example : MonoBehaviour
    {
        private TextMeshProUGUI textMeshProUGUI;

        private string player = "fe1zed";
        private int level = 999;
        private int skillPoints = 100;
        private string quest = "Make coding easier!";

        private string[] languages = { "English", "French", "German", "Russian", "Spanish" };
        private int selectedLanguageIndex = 0;

        // Subscribe to language updates
        private void OnEnable()  => LocalizationSystem.OnLanguageChanged += OnLanguageChanged;
        private void OnDisable() => LocalizationSystem.OnLanguageChanged -= OnLanguageChanged;

        private void OnLanguageChanged() => SetLocalizedText();

        private void SetLocalizedText()
        {
            if (textMeshProUGUI == null) textMeshProUGUI = GetComponent<TextMeshProUGUI>();

            // Example 1 — built-in interpolation
            // CSV key: example_complex,{0}, your character is level {1}. You have {2} skill points and your current quest is: {3}.
            textMeshProUGUI.text = LocalizationSystem.GetLocalisedValue("example_complex",
                player,      // {0}
                level,       // {1}
                skillPoints, // {2}
                quest        // {3}
            );

            // Example 2 — standard string interpolation
            // textMeshProUGUI.text = $"{LocalizationSystem.GetLocalisedValue("save_game")} | Yes/No";
        }

        private void Update()
        {
            // Space: increment level and refresh text
            if (Input.GetKeyDown(KeyCode.Space))
            {
                level++;
                SetLocalizedText();
            }

            // L: cycle through languages
            if (Input.GetKeyDown(KeyCode.L))
            {
                LocalizationSystem.SetLanguage(languages[selectedLanguageIndex]);
                selectedLanguageIndex = (selectedLanguageIndex + 1) % languages.Length;
            }
        }
    }
}

If a key has missing arguments the output returns the unmodified value. If a key is missing entirely, the key string itself is returned as fallback.

Auto-Translate

The Auto-Translate window generates translations using external API providers. Open it from:

Tools → Localization → Auto Translate    (Ctrl+Alt+A)

Usage:

  1. Select the source language file
  2. Select the target language file
  3. Choose a translation provider and enter your API key
  4. Click Generate — translations are previewed before being applied
  5. Review results, then click Apply to write to the CSV

Supported providers:

  • DeepL — requires a DeepL API key
  • Microsoft Azure Translator — requires an Azure API key and region

Only localization values are translated — keys remain unchanged. Existing translations can be preserved or overwritten. Large datasets are processed in batches to comply with provider rate limits.

API keys are not included. An active internet connection is required. Always review generated translations before shipping — automated translations may contain errors.

Script Reference

LocalizationSystem

Namespace: fe1zed.InstantLocalization — static class, central API for all language state management.

// Property — currently loaded language code
public static string Language { get; private set; }

// Event — fires whenever a new language loads
public static event Action OnLanguageChanged;

// Load a language by code (must match CSV filename)
public static void SetLanguage(string languageCode);

// Get a translated string for the current language
public static string GetLocalisedValue(string key);

// Reload the current language from CSV (useful after manual edits)
public static void ReloadLanguage();

// Force all LocalizeText components to refresh
public static void UpdateTexts();

LocalizeText

Namespace: fe1zed.InstantLocalization — MonoBehaviour, attach to any GameObject that has a TextMeshProUGUI component.

// The key used to look up text in the active language CSV
public string localizedTextID;

Automatically subscribes to OnLanguageChanged in OnEnable and unsubscribes in OnDisable. Updates the attached TextMeshProUGUI whenever the language changes, both in the Editor and at runtime.

Editor Tools Menu

All tools are located under Tools → Localization:

  • Open Localization Window Ctrl+Shift+T — opens the Key Selector Window
  • Update Texts — forces all LocalizeText components to refresh; useful after manual CSV edits
  • Auto Translate Ctrl+Alt+A — opens the Auto-Translate Window

The Key Selector Window, CSV Manager, Favorites system, and Auto-Translate Window are Editor-only and are not included in builds.

Best Practices

  • Use the Editor Window — manage keys through the Key Selector Window instead of manually editing CSV files to ensure consistency across all language files.
  • Use Favourites — mark frequently used keys (common UI labels, buttons) as favourites to speed up selection.
  • Consistent naming — use structured key names like ui_button_start or menu_settings_title. Consistent naming improves maintainability as the project grows.
  • Test language switching frequently — regularly switch languages during Play Mode to verify all components update correctly and no keys are missing.
  • Review auto-translations — always review machine-generated translations for accuracy and context before release.
  • Back up CSV files — keep backups before performing bulk edits or automated operations.
  • Test in builds — localization files are accessed differently in the Editor (direct file access) vs runtime (Resources.Load). Always test in a built player.
  • Keep CSVs clean — use UTF-8 encoding and avoid duplicate keys, malformed rows, or unintended formatting changes from spreadsheet software.

Troubleshooting

  • Text doesn't update when switching languages — ensure LocalizeText is attached; the key exists in all CSV files; use Tools → Localization → Update Texts to force a refresh.
  • CSV file not found — verify the file is in Assets/Instant Localization/Resources/ and the filename matches the language code being loaded.
  • Favourites not persisting — check that Assets/Instant Localization/Data/InstantLocalizationFavorites.asset exists; it is auto-created on first use.
  • Keys appear as the key string — the key exists in one language file but not others; use Edit in the Key Selector Window to add the missing translations.
  • Auto-Translate empty or incomplete results — verify your API key is valid, the provider is reachable, your internet connection is active, and check quota or billing limits.
  • DeepL AuthenticationFailed despite valid key — ensure the key uses header-based authorization (Authorization: DeepL-Auth-Key); legacy form-body auth is deprecated.

Technical Notes

  • Performance — CSV files are loaded once when the language is set. Runtime lookups use Dictionary<string, string> for O(1) performance.
  • Thread safety — the system is not thread-safe. Always call methods from the main Unity thread.
  • File encoding — CSV files must be UTF-8 encoded to support special characters and non-Latin alphabets.
  • Editor vs runtime — Editor tools access CSV files directly from the project folder; at runtime the system uses Resources.Load. Always test localization in a built player.
