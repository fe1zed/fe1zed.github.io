Dev Console Pro Documentation

v1.3.0 · Unity 2020.3+

Installation

Import the package from the Unity Asset Store. No additional dependencies are required.

  1. Open the Package Manager: Window → Package Manager
  2. Switch to My Assets and find Dev Console Pro
  3. Click Import

Drag the DevConsolePro prefab from DevConsolePro/Prefabs/ into your scene. The prefab is self-contained — it creates its own Canvas and persists across scene loads.

Quick Start

Press F1 at runtime to open and close the console. Type help to list all registered commands.

Add [ConsoleCommand] to any public method and wrap the class with [DevConsole("prefix")]:

using DevConsolePro;
using UnityEngine;

[DevConsole("player")]
public class PlayerController : MonoBehaviour
{
    [ConsoleCommand("Teleport the player to a position")]
    public void Teleport(float x, float y, float z)
    {
        transform.position = new Vector3(x, y, z);
    }
}

Open the console at runtime and type:

player.teleport 0 5 0

Commands

Use [ConsoleCommand] on any public method inside a class marked with [DevConsole].

Simple form

[ConsoleCommand("Description")]
public void MyMethod(float value) { }

// With a shortcut alias
[ConsoleCommand("Description", shortcut: "sc")]
public void MyMethod(float value) { }

Explicit prefix and aliases

[ConsoleCommand("myprefix", "Description", "alias1", "alias2")]
public void MyMethod() { }

The resulting command name is prefix.methodName (lowercase first letter). If no prefix is provided on the attribute, the class-level [DevConsole] prefix is used. If neither provides a prefix, the method name alone is used.

Variables

Use [ConsoleVariable] on any public field or property. Variables can be read and written from the console.

[DevConsole("player")]
public class PlayerController : MonoBehaviour
{
    [ConsoleVariable("Movement speed")]
    public float speed = 5f;
}
player.speed          // prints current value
player.speed 12.5     // sets value to 12.5

Read-only properties (no setter) can only be read — attempting to set them logs a warning.

Explicit prefix and aliases

[ConsoleVariable("myprefix", "Movement speed", "spd", "velocity")]
public float speed = 5f;

Auto-Register

Applying [DevConsole] to a class automatically registers all public fields and properties as variables — no [ConsoleVariable] needed on each one.

[DevConsole("player")]
public class PlayerStats : MonoBehaviour
{
    public float health  = 100f;  // → player.health
    public float stamina = 80f;   // → player.stamina
    public bool  isAlive = true;  // → player.isAlive
}

Use [ConsoleIgnore] on any member you want to exclude from registration.

[DevConsole("player")]
public class PlayerController : MonoBehaviour
{
    public float speed = 5f;          // registered → player.speed

    [ConsoleIgnore]
    public float internalTimer = 0f;  // skipped
}

Log Capture

DevConsole Pro hooks into Application.logMessageReceived automatically. Every Debug.Log, Debug.LogWarning, and Debug.LogError call in your project appears in the console output in real time — no extra setup needed.

[12:34:56]  > player.speed 20           ← command echo
[12:34:56]  Player speed changed        ← Debug.Log
[12:34:57]  Health is low!              ← Debug.LogWarning
[12:34:58]  NullReferenceException...   ← Debug.LogError

Static Members

Commands and variables don't need to be on a MonoBehaviour. Static members are fully supported — no scene instance is required.

[DevConsole("game")]
public static class GameSettings
{
    [ConsoleVariable("Master volume")]
    public static float masterVolume = 1f;

    [ConsoleCommand("Quit the application")]
    public static void Quit()
    {
        Application.Quit();
    }
}
game.masterVolume 0.5
game.quit

Aliases

Both [ConsoleCommand] and [ConsoleVariable] accept aliases — alternative names that map to the same command.

[ConsoleCommand("player", "Teleport to position", "tp", "warp")]
public void Teleport(float x, float y, float z) { }

All three names work identically at runtime:

player.teleport 0 5 0
tp 0 5 0
warp 0 5 0

ConsoleArgs

For commands that need variable argument counts or custom parsing, use ConsoleArgs as the sole parameter instead of typed values.

using DevConsolePro.Core.Commands;

[ConsoleCommand("Print all provided arguments")]
public void Echo(ConsoleArgs args)
{
    for (int i = 0; i < args.Count; i++)
        Debug.Log(args[i]);
}
MemberReturns
args[i]string — raw token
args.GetString(i)string with bounds check
args.GetInt(i)int (InvariantCulture)
args.GetFloat(i)float (InvariantCulture)
args.GetBool(i)bool
args.CountTotal argument count
args.ToArray()Copy of all tokens as string[]

Enum and bool parameters show autocomplete suggestions automatically. bool variables offer true / false; enums offer every value in the type.

public enum Difficulty { Easy, Normal, Hard }

[DevConsole("game")]
public class GameManager : MonoBehaviour
{
    [ConsoleCommand("Set game difficulty")]
    public void SetDifficulty(Difficulty difficulty)
    {
        Debug.Log($"Difficulty set to {difficulty}");
    }
}
game.setDifficulty Hard

Runtime Registration

Commands can be registered at runtime without attributes using DevConsoleAPI.Register. Useful for procedurally generated commands, plugin systems, or anything not known at compile time.

No-argument command

DevConsoleAPI.Register("scene.reload", () => SceneManager.LoadScene(0));

// With description (shown by 'help')
DevConsoleAPI.Register("scene.reload", "Reload the active scene", () => SceneManager.LoadScene(0));

Typed arguments

Generic overloads provide automatic ghost hints and argument type display. Parameter names are read from the lambda automatically.

// Single argument — hint shows: time.scale <float:value>
DevConsoleAPI.Register<float>("time.scale", "Set time scale", (float value) =>
{
    Time.timeScale = value;
});

// Two arguments — hint shows: player.move <float:x> <float:z>
DevConsoleAPI.Register<float, float>("player.move", "Move player on XZ", (float x, float z) =>
{
    player.transform.position = new Vector3(x, player.transform.position.y, z);
});

// Three arguments
DevConsoleAPI.Register<float, float, float>("cam.pos", "Set camera position",
    (float x, float y, float z) => cam.transform.position = new Vector3(x, y, z));

Overloads exist for 1 through 4 typed parameters. Passing an unsupported type throws immediately at registration time. For 5+ arguments, pass an explicitly typed delegate:

DevConsoleAPI.Register("cmd", "Description",
    (Action<int, float, string, bool, int>)MyFiveArgMethod);

Raw arguments

DevConsoleAPI.Register("log.print", (ConsoleArgs args) =>
{
    Debug.Log(args.GetString(0));
});

Unregistering

Always unregister commands when the owning object is disabled or destroyed to avoid dangling handlers.

private void OnEnable()
{
    DevConsoleAPI.Register("fx.enable", OnFxEnable);
}

private void OnDisable()
{
    DevConsoleAPI.Unregister("fx.enable");
}

Unregister returns true if the command was found and removed, false if it didn't exist.

API Access

Use DevConsoleAPI to execute commands from code — useful for cheat systems, automated tests, or custom UIs.

using DevConsolePro.Core;

DevConsoleAPI.Execute("player.speed 20");
DevConsoleAPI.Execute("game.setDifficulty Hard");

DevConsoleAPI.Initialize() is called automatically the first time you call Execute(). You can also call it explicitly at startup:

private void Awake()
{
    DevConsoleAPI.Initialize();
}

OnCommandExecuted

Subscribe to DevConsoleAPI.OnCommandExecuted to react after any command executes successfully. The event does not fire if the command throws an exception.

private void OnEnable()
{
    DevConsoleAPI.OnCommandExecuted += OnCommandRan;
}

private void OnDisable()
{
    DevConsoleAPI.OnCommandExecuted -= OnCommandRan;
}

private void OnCommandRan(string commandName, string[] args)
{
    string argsDisplay = args.Length > 0 ? string.Join(", ", args) : "none";
    Debug.Log($"Executed '{commandName}' with args: {argsDisplay}");
}

Common uses: analytics tracking, CI test runners, achievement triggers.

Example — mirror output to a custom UI

private void OnEnable()
{
    ConsoleLogService.LogsChanged += RefreshUI;
}

private void OnDisable()
{
    ConsoleLogService.LogsChanged -= RefreshUI;
}

private void RefreshUI()
{
    foreach (ConsoleLogEntry entry in ConsoleLogService.Entries)
        Debug.Log($"[{entry.Timestamp:HH:mm:ss}] {entry.Message}");
}
